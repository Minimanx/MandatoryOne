RESTful API

About

REST is an architectural style described by six constraints.
The constraints are pretty lengthy and complicated, so I won't get into those. You can read about them here.
RESTful APIs takes requests of the HTTP CRUD methods to retrieve, modify or create data on the server.
The most common ones are:

  • GET

  • POST

  • PUT/PATCH

  • DELETE

We can also add data to the body of requests, which are then used on our server in whatever way we choose.
Additionally, it is also possible to send data through the URL in 2 different ways:

  • Path Variable

  • Query String

A path variable is information sent after a / in the url as such: www.api.com/apis/1 (the 1 being the variable).
They are usually used to identify a specific resource, via for example, an ID.
On the other hand, query strings are usually used for filtering when you're expecting multiple results: www.api.com/apis?name=cool (?name=cool being the query).
This looks at our list and returns all apis that have the name "cool" or contains "cool". That depends on how you decide to implement it.

Typical Endpoints for a RESTful API

Method Endpoint Result
GET /movies Retrieves all movies
GET /movies/:id Retrieves a movie with the given id
POST /movies Creates a new movie
PUT /movies/:id Updates a movie with the given id
PATCH /movies/:id Partially updates a movie with the given id
DELETE /movies/:id Deletes a movie with the given id
